# D-037: ゲーム内ポーズ(ESC)とやり直し

## 決定

1. フェーズに `Paused` を追加する。遷移規則は GameFlow が持つ:
   `PauseGame`(Playing → Paused)/ `ResumeGame`(Paused → Playing、盤面そのまま)/
   `RestartGame`(Paused → Playing、盤面リセット)。
   ESC はトグル(Playing でポーズ、Paused で解除。Godot 層は PauseGame が
   不成立なら ResumeGame を試すだけで、状態判定は GameFlow に委ねる)
2. ポーズ中の UI は Declaree で宣言する:「ポーズ中」ラベル+[続ける][やり直す][終了]。
   [続ける] → `ResumeGameCommand`、[やり直す] → `RestartGameCommand`、
   [終了] → 既存の `ExitGameCommand`(D-032 の配線と同型)
3. やり直しの盤面リセットは責務を分ける:
   - `SuikaLogic.Reset()` … フルーツ全消去・スコア 0・ゲームオーバー解除。
     **乱数系列と ID 採番は継続する**(決定論の再現性と ID の追跡可能性を優先)
   - Godot 層は `GameCommandRouter.RestartRequests`(遷移成立時のみ通知)を購読して
     物理ボディを破棄し `Reset()` を呼ぶ
4. 物理停止は `GetTree().Paused = TimeControl.IsPaused || Phase == Paused`。
   Statee の時間制御(D-026)とゲーム内ポーズは独立した停止要因として論理和を取る
5. 実入力経路のキー注入コマンド `key`(`--arg key=escape` 等)を追加する
   (GUIDELINE 3.2。click と同じく Viewport.PushInput で press/release を注入)

## 背景

- ユーザー要求: ESC でポーズ、ポーズ中は[やり直す][終了]、
  [やり直す]でフルーツとスコアをリセットして再開、[終了]でゲームを閉じる。
  追加要求: [続ける]でポーズ解除、ESC 再押下でも解除
- ポーズ中もボタンが押せるのは Main の `ProcessMode = Always` が UI に継承されるため

## トレードオフ

- 乱数系列を継続するため「やり直し後の抽選列」は初回プレイと一致しない。
  完全な初期化(シードの再適用)が要るなら SuikaLogic を作り直す設計に変える
- ゲーム内ポーズ中は物理フレームが進まないため `wait` は成立しない
  (Statee の pause と同じ性質)

## 状態

採用。ロジックはユニットテスト(GameFlow / Router / Reset で37件緑)、
Godot 層は headless E2E で確認済み(ESC → メニュー表示 → やり直しでスコア0・
フルーツ0・再投下可 → 終了で exit 0。続けるクリック / ESC 再押下での解除は
盤面・スコアが維持される)。
