# D-077: CLI に人間向けの watch サブコマンドを追加

## 決定

- `Statee.Cli` に `watch` サブコマンドを追加する。指定した State パスを TCP 経由で
  定期取得(既定 0.5 秒間隔)し、TOON をコンソールに表示し続ける
- 表示は人間の目視確認用。TTY では画面クリア+ヘッダ付き再描画、リダイレクト時は
  タイムスタンプ区切りで追記出力する
- `--count N` で有限回にできる(0 = 無限、Ctrl+C で終了)。終了コードは最後の取得の成否
- `watch` の stdout は表示用であり、既存コマンドの「stdout = payload のみ」契約の対象外とする

## 背景

- TOON 文字列を人間が確認する手段が「CLI を手で連打する」しかなかった
- Godot 内にデバッグ表示を作る案もあったが、headless 前提の主用途では画面が存在せず、
  ゲームごとのデバッグ UI 実装(D-013 の逆流)にもなるため、既存の pull 型
  スナップショット経路(TCP)に薄いクライアントを足す方を選んだ

## トレードオフ

- 1取得=1接続を interval ごとに繰り返すため、AI(CLI)との同時利用でも接続を
  占有しないが、常時接続の push 型より遅延・オーバーヘッドはある(目視用途では十分)
- 差分ハイライト等の整形は持たない。必要になったら別途検討する

## 状態

採用(PingTarget に対する E2E で正常系・エラー表示・終了コードを確認済み)
