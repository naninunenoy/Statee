# D-078 dotee を複数フレーム(スプライトシート)対応にする

## 決定

`.sprite.txt` に `frame: 名前` 節を導入し、1 ファイルで複数フレームを定義できるようにする。
render は全フレームを**横並びに合成した 1 枚の PNG(スプライトシート)**として出力する。

- 形式: `palette:` 節のあと、`grid:`(従来の単一フレーム)**または** `frame: 名前` の繰り返し。
  混在はエラー。全フレームは同一寸法。フレーム名の重複・空名はエラー
- フレームの並び順 = ファイル内の定義順 = シート内の左からの順。
  Godot 側は `Sprite2D.Hframes`(フレーム数)+ `Frame`(定義順の添字)で参照する。
  フレーム数と名前一覧は render 時にコンソールへ出力する
- アニメーションもタイルセットも「同寸法フレームの並び」として同じ仕組みに載せる
  (タイル専用の形式は作らない)

## 背景

- D-070 は dotee のスコープを「テキスト → PNG 変換器」に留め、
  アニメーションは「必要になるまで作らない」としていた
- MessBreak のグラフィック拡充(マップタイル・敵アニメ・キャラ歩行・設置物の状態差分)で
  複数フレームが実際に必要になった。1 フレーム = 1 ファイルの運用は
  数十フレーム規模ではパレットの重複と管理コストが大きい

## トレードオフ

- シートは横 1 列固定(格子配置なし)。フレーム数が極端に多いと横長になるが、
  ドット絵サイズでは実害がなく、Godot の Hframes 参照が最も単純になる
- フレーム名は PNG に残らない(定義順が正)。名前→添字の対応はゲーム側コードが
  .sprite.txt の定義順に合わせて持つ
