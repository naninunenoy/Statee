# D-063 キーボード/ゲームパッド操作は Godot 標準フォーカスに乗る(掴みモード並び替え・フォーカストラップ)

## 決定

UI のキーボード/ゲームパッド操作は、フォーカスを IR で宣言管理せず **Godot 標準の
フォーカスシステム(`ui_*` アクション+自動推定ナビゲーション)に乗る**。
Declaree が足すのは次の4点だけ。

- **`UiNode.AutoFocus`** — 生成されてツリーに入った時点で一度だけフォーカスを掴むヒント。
  フォーカスの現在位置はエンジン管理の一時状態なので宣言しない(宣言すると
  ユーザーのナビゲーションと再宣言が喧嘩する)
- **Reconcile のフォーカス復元** — パッチ経路は Control を破棄しないので自然に生存。
  Rebuild 経路は破棄前にフォーカス中の Name を控え、再構築後に同名へ `GrabFocus()`。
  サブツリー内で見つからない場合(並び替え等)は Reconcile ルートがツリー全体から復元する
- **Overlay のフォーカストラップ** — Stack 内で Overlay より背面の子に
  `FocusBehaviorRecursive = Disabled` を設定(マウスは veil の MouseFilter.Stop、
  キーボードはこれが遮断する)
- **State への読み出し公開** — UiSnapshot がフォーカス中の要素に `focused: true` を採取する。
  「宣言はしないが観測はできる」。エージェントはキーボード操作の結果を ui/tree で検証できる

ReorderList の並び替えは**掴みモード**: フォーカスした行で掴む(既定 Ctrl+Space /
パッド X。InputMap 未定義ならコンテナが既定バインドを登録)→ ui_up / ui_down で
ドロップ先を移動 → ui_accept で確定 / ui_cancel で中止。判定は純C#の
`ReorderKeyboard` 状態機械が持ち、マウスドラッグと同じ3イベント
(update / commit / cancel。D-062)へ写像するため、DraggingIndex / DropIndex の
宣言と可視化をそのまま共有する。掴み中の方向キー等は消費し、フォーカス移動に漏らさない。

あわせて標準 `key` コマンドが `ctrl+space` 形式の修飾キー付き注入に対応した。

## 背景

- TodoApp をキーボード/パッドだけで操作可能にする要求。Godot は
  ①物理デバイスを `ui_*` アクションへ抽象化、②Viewport が単一のフォーカスを管理し
  方向アクションで遷移、③ui_accept をマウスと同じ `pressed` シグナルへ合流、
  という構造なので、Declaree のイベントディスパッチは無変更で成立する
- フォーカス完全 IR 管理案は、エンジンの焦点機能の再実装とユーザー操作との
  競合処理が重く、見返り(宣言との一貫性)に対して割に合わないため不採用
- ゲームパッドは `ui_*` の既定バインドに最初から含まれるため、キーボード対応が
  そのままパッド対応になる(D-043 のアクション単位の原則と同型)

## トレードオフ

- フォーカスの遷移先(どの順に巡るか)は Godot の自動推定に依存する。
  変な遷移が出たら focus_neighbor の明示指定を IR に足す(必要が出てから)
- Rebuild 時の復元は Name 頼み。同名要素が消えた場合(行削除等)はフォーカスが
  失われたままになる(近傍へのフォールバックは必要が出てから)
- Overlay のトラップは Stack 直下の兄弟にだけ効く(TodoApp の構造では十分)。
  Overlay.Visible の切り替えでは効かず、出し入れ(子の増減)で使う前提
- 掴みアクションの既定バインド(Ctrl+Space / パッド X)は Declaree.Godot が
  InputMap へ登録する。ゲーム側が同名アクションを定義済みなら触らない
- E2E はキーボード注入のみで検証(パッドは同じアクションに合流するため)
