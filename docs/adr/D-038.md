# D-038: UI 要素の安定 Name と name 指定クリック

## 決定

1. Declaree の UiNode に共通プロパティ `Name` を追加する。記述子の `name` として
   `ui/tree` に公開され、UiRenderer は Godot ノード名にも反映する(GUIDELINE 3.4 の安定 ID)
2. 記述子ツリーの検索 `UiTree.FindByName`(深さ優先、なければ null)を Declaree に置く
3. SuikaGame の `click` コマンドは `--arg name=StartButton` を受け付ける。
   `ui/tree` スナップショットを FindByName で引き、Rect の中心座標を導出して
   **従来と同じ実入力経路(PushClick)** でクリックする。座標指定(`x=,y=`)も残す
4. AI の操作規約: name は「何をする要素か」が想像できる命名にし(StartButton 等)、
   explain(D-032)と合わせて要素の意味を伝える。座標決め打ちはしない

## 背景

- D-036 のトレードオフで「旧 game/ui の安定 Id が失われた。要素をパスで指名する
  語彙が必要になったら足す」と保留していた。ユーザー要求
  (「UI要素には name を与えて name から操作するように」)で必要が確定した
- name 解決後も InputEvent 注入を通すため、非表示・無効・画面外の要素には
  正しく「効かない」という D-031 スライス③の性質は保たれる

## トレードオフ

- name の一意性は宣言側の規律に委ねる(同一ツリー内の重複は先勝ち)。
  ツリーは小さく、重複が問題になったら Describe で検証を足す
- name → Rect 解決はスナップショット(前物理フレーム)基準。再構築直後の
  1フレームは Rect が古い・未確定のことがある(エラー応答で観測できる)

## 状態

採用。Declaree はユニットテスト(16件緑)、SuikaGame は headless E2E で確認済み
(name 指定クリックだけでタイトル → プレイ → ポーズ → やり直し → 終了 exit 0。
未知の name は「UI 要素が見つからない」エラー)。
