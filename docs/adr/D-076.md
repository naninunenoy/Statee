# D-076 ゲーム内ポーズと tick コマンドの関係(素通りを仕様とする)

## 決定

ゲーム内ポーズ(人間向けのポーズメニュー等)が論理 tick を止めていても、
エージェントの `tick` コマンド(D-072)は**ポーズを素通りしてロジックを直接進めてよい**。
これを仕様とする。あわせてゲームは、ポーズ状態を State に公開する
(例: MessBreak の `Paused`)。

## 背景

- MessBreak に Esc ポーズメニューを実装した際、「人間のポーズ中にエージェントが
  盤面を進められる」ことをどう扱うかが未整理だった
- Statee の時間制御は freeze(TimeControl)が担っており、`tick` コマンドは
  「freeze 中でも入力つきで決定的に進める」ための検証経路。ゲーム内ポーズは
  **人間の入力経路に対する演出・UX** であり、検証経路とは層が違う

## トレードオフ

- 「ポーズの間は世界が完全に止まる」ことを検証したいシナリオは、`Paused` State を
  読んで自分で tick を送らない、という規約ベースの尊重になる(強制はしない)
- ロジック側でポーズを表現したいゲーム(ポーズ中も進む要素があるなど)は、
  ポーズをロジックの状態として実装すればよく、本決定はそれを妨げない

## 状態

採用。MessBreak が最初の適用例(Paused を State に公開、tick は素通り)。
