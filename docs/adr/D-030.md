# D-030 決定記録を docs/adr 下に1決定1ファイルで分割

- **決定**: 旧 docs/MEMO.md(D-xxx を1ファイルに追記)と旧 docs/NOTES.md(雑メモ)を廃止し、
  `docs/adr/` 配下に移す。
  - 決定は `docs/adr/D-0xx.md`(1決定=1ファイル)。タイトルはファイル内の見出しに持たせ、
    ファイル名は番号のみにする(コードコメントからの参照を短く安定に保つ)
  - 索引は `docs/adr/README.md`。新しい決定を足したら索引にも1行追加する
  - 雑メモは `docs/adr/notes/<topic>.md`。昇格の運用(確定したら D-xxx へ)は従来どおり
- **背景**: MEMO.md が29決定・450行超に成長し、特定の決定を読む・更新するたびに
  全体を読み込むコストが増えた。1ファイル化で読み書きが決定単位になり、
  diff・レビューも決定ごとに閉じる。
- **トレードオフ**:
  - (+) 決定単位の読み書き・参照(`docs/adr/D-018.md`)ができる
  - (+) 索引で全体を一覧でき、番号の採番ミスも索引で気づける
  - (−) 索引 README の手動メンテが1手間増える
  - (−) 横断検索は `grep -r docs/adr` が必要(1ファイル時代は1ファイル読めばよかった)
- **状態**: ✅ 移行完了(全29決定 + notes 3件。コード内の参照も docs/adr へ更新)
