# D-059 RaidBossをリアルタイム化(自動Tick+レーン移動+予告付きボス攻撃)

## 決定

RaidBoss を「入力したときだけTickが進むターン制」から「一定間隔でTickが自動進行する
リアルタイム制」へ変更し、移動と回避を導入する。

- **自動Tick**: RaidBoss.Godot が `TickIntervalSeconds`(0.5秒)ごとに自動でTickを進める。
  キー入力は「次のTickで実行する行動」として蓄え、自動Tick時に消費する。
  ネット対戦では自分の行動を自動送信し、確定は従来どおりサーバの入力揃い待ち(D-054)。
  freeze 中は自動Tickが止まり、`step` コマンドによる手動進行だけになる(シナリオ検証は
  freeze してから従来どおり決定論的に進める)
- **レーン移動**: プレイヤーは `LaneCount`(7)本のレーン上を `MoveLeft` / `MoveRight` で
  1Tickに1つ移動できる(端で止まる)。初期位置は中央寄りに等間隔
- **予告付きボス攻撃**: ボスは `BossAttackInterval`(5Tick)ごとに対象レーンを予告し、
  `BossAttackWindupTicks`(2Tick)後にそのレーンへ着弾する。着弾時に予告レーンにいる
  プレイヤー全員が被弾する(移動していれば回避できる)。対象レーンは
  `attackNumber % LaneCount` の決定論的巡回で選ぶ(乱数不使用。シード差異による
  ロックステップ崩壊を避ける)
- 回避可能になったため `BossAttackDamage` を 15→40 に引き上げた(被弾3発でダウン)
- 行動は1Tickに1つ(`Idle` / `Attack` / `MoveLeft` / `MoveRight`)。ワイヤ表現は
  `action=idle|attack|left|right`
- State に `PlayerLanes`(カンマ区切り)と `PendingBossAttack`(「レーン:残りTick」、
  なければ空文字)を追加公開。Godot の描画は予告レーンの赤帯(着弾が近いほど濃い)と、
  自動Tickの経過割合による弾の補間(Tick間も滑らかに飛ぶ)を追加した

## 背景

- D-058 まではシューティング「演出」であって、時間はキーを押したときだけ進み、移動も
  存在しなかった。ユーザーから「移動できないし、Qを押すと黄色の点が出るだけ」という
  指摘を受け、リアルタイムに弾が飛び交い回避に意味があるゲームへ進める判断をした
- 時間の自動進行はロックステップ(D-054)と両立させる必要がある。「入力がないTickは
  Idleを自動送信する」ことで、サーバの確定機構(TickBundleAuthority)は一切変えずに
  リアルタイム化できた
- ロジック層は引き続き外部Tick駆動・壁時計禁止(D-023)。壁時計(0.5秒間隔)を知るのは
  Godot 層だけで、GameLogic は従来どおり `Step(actions)` を呼ばれた回数しか知らない

## トレードオフ

- 自動Tickで入力が毎秒流れるため、Idle の送信・確定はログに残さないようにした
  (ログの信号雑音比を保つ)。State の TickCount は放置でも増え続ける
- クライアント間で送信タイミングがずれると、確定Tickの進行は「最も遅いクライアント」に
  律速される(ロックステップの性質。今回は許容)
- シナリオ検証(network-fullgame.rb)は freeze してから step で進める形へ書き換えが
  必要だった。wait コマンドは freeze 中も1フレームずつ進めるため、理論上は待機中に
  自動Tickが1回混入し得るが、Victory/Defeat 後の Step は no-op なので検証結果には
  影響しない
