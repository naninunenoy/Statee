# D-074 State スナップショットの自動生成(Generator 拡張)は見送り

## 決定

「ロジック型を指定したら public プロパティを自動で State 化する」Generator 拡張は
**当面作らない**。GameState の Snapshot record + `[StateeField]` の手書きを続ける。

## 背景

- GameState はロジックの公開プロパティとほぼ 1:1 の写経で、フィールド追加のたびに
  logic / Snapshot / StateeField の 3 箇所を触る摩擦があった
  (docs/adr/notes/game-dev-feedback-messbreak-slice1.md の項目 2)
- しかし実例(MessBreak)を見ると、スナップショットには機械的に導出できない判断が
  本質的に含まれる: null の既定値化(`mob?.Hp ?? 0`)、Vector2 の X/Y への平坦化、
  enum の文字列化、リストの件数への縮約など。全自動生成はこれらの表現を
  属性のオプションとして再発明することになり、手書きより複雑になる

## トレードオフ

- 写経の摩擦は残る。ただし「State には検証に必要な情報を全公開する」という設計行為
  そのものであり、フィールド追加時に 3 箇所触るのは意図の表明でもある
- 将来、同型のゲームが増えて平坦化・縮約のパターンが安定したら、
  部分的な省力化(Vector2 の平坦化サポート等)から再検討する

## 状態

見送り(再検討条件つき)。
