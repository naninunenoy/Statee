# D-047: Godot 側の Statee 配線イディオムを libs/Statee.Godot に共通化する

## 決定

1. `src/`・`game/` と同階層に **`libs/`** を新設し、ゲーム開発ヘルパーライブラリを置く
2. 初のライブラリとして **`libs/Statee.Godot`**(Godot 依存)を作り、両サンプルと
   /new-game テンプレートで3重に複製されていた Statee 配線の定型を移す:
   - `StandardCommands.Register` … ping / key(PushInput)/ screenshot / quit の一括登録
   - `KeyBinding` + `KeyBindingTable` … キーバインド表(D-039)。`TryHandle`(入力処理)と
     `CreateInputStateProvider`(game/input State)を同じ表から導出。
     場面フィルタは `ActiveIn`(表示名)+ `IsActive`(判定)で任意対応
   - `CmdlineArgs.ParseInt` … `--seed=` / `--port=` 起動引数
   - `StateeLogging.CreateLoggerFactory` … LogBuffer + コンソールの定型ロガー
3. Godot 非依存の `SnapshotStateProvider` は `src/Statee.Core` 本体へ移す
4. 依存方向は `game/ → libs/ → src/` の一方通行。`src/` は Godot 非依存を保つ

## 背景

- SuikaGame・RogueGame・/new-game テンプレートの3箇所に、文字単位でほぼ同一の
  配線コードが存在した(「3回書いたら共通化」の基準を満たした)
- このリポジトリの目的は Statee と「ゲーム開発ヘルパーライブラリ」の2本柱だが、
  後者の置き場が無かった。`src/` に入れると「フレームワーク本体は Godot 非依存」の
  境界(D-013 の精神)が濁るため、独立トップディレクトリとする(`declaree/` と同格)
- /new-game テンプレートが縮み、skill が古びる表面積が減る(D-045 のトレードオフ緩和)

## トレードオフ

- Statee.Godot は Godot 依存のため xUnit での単体テストが困難。当面は
  両ゲームのビルド+E2E(`tools/build-all.ps1` と /verify)が実効テスト
- 共通化の対象は「機械的に同一の配線」のみ。HUD 描画・State クラス・ゲーム固有
  コマンドはゲームごとに書く(継承ベースの基底クラスは作らない。コンポジションで使う)
- SuikaGame には従来 ping が無かったが、StandardCommands 経由で追加された
  (標準コマンドの統一という意味で意図した変化)

## 状態

採用。
