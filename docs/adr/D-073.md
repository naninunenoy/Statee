# D-073 起動引数 --frozen(起動直後から freeze)

## 決定

ターゲットの起動引数に **`--frozen`** を追加する。指定時はゲームループ開始前に
`TimeControl.Freeze()` を呼び、tick 0 の凍結状態で待ち受けを開始する。

- 判定は `CmdlineArgs.HasFlag("--frozen")`(libs/Statee.Godot に HasFlag を追加)
- 適用はターゲット側の `_Ready` の責務(雛形 /new-game に組み込み。
  ShootingGame・MessBreak は対応済み)

## 背景

- headless 起動から CLI で freeze するまでの間に実時間で tick が進み、
  seed を注入しても接続タイミングで盤面が変わっていた
  (docs/adr/notes/game-dev-feedback-messbreak-slice1.md の項目 3)。
  `--frozen` により再現シナリオを tick 0 から決定論的に書ける

## トレードオフ

- フレームワーク強制ではなくターゲット実装の規約(TimeControl の配線自体がゲーム側の
  ためここだけ強制できない)。未対応ターゲットではフラグが無視される。
  verify スキルに「対応しているターゲットのみ」と明記した

## 状態

採用。MessBreak / ShootingGame / 雛形テンプレートに適用済み。
