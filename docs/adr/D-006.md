# D-006 ID 等の ValueObject は UnitGenerator

- **決定**: [UnitGenerator](https://github.com/Cysharp/UnitGenerator) で ObjectId 等の型を生成する。
- **背景**: AI がフレームを跨いで同一オブジェクトを追跡するため、ID の型安全性と安定性が重要。
- **補足(D-024 の FruitId を受けて)**: ID の型は層で分ける。
  - ゲームロジック内はドメイン固有の型(例: `FruitId`)。概念ごとに型を分けることで
    取り違えをコンパイルエラーにするのが UnitGenerator を使う目的なので、
    ロジック内を汎用 `ObjectId` に統一はしない
  - 「AI から見た安定 ID」(GUIDELINE.md §7)はフレームワークの語彙。オブジェクト粒度
    State を実装するスライスで汎用 `ObjectId` を Statee.Core に導入し、
    Godot 統合層でドメイン ID → `ObjectId` に写す(変換は境界の一行)。
    使う実装が無い今は導入しない(YAGNI。D-019 の見送りと同じ理由)
