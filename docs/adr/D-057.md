# D-057 マッチング待機は1人でも開始可能。HP0は一時的な操作不能(全滅でDefeat)

## 決定

- `GameLogic.MinPlayerCount` を2から1へ変更し、ロビーで1人しかいなくても
  部屋作成者の `start` でPlayingへ遷移できるようにする(ソロプレイを許容)
- プレイヤーのHPが0以下になった瞬間、`IncapacitationDuration`(3Tick)だけ
  操作不能になる(その間の入力はIdle扱いで無視される)。操作不能が明けると
  `ReviveHp`(PlayerMaxHpの半分)まで回復して復帰する
- 全プレイヤーのHPが同時に0以下になった場合のみDefeatへ遷移する(誰か1人でも
  行動可能なら試合は続く)。ソロプレイでは「全員」が自分1人なので、
  HP0=即Defeatになる(復帰の機会がないまま試合終了。仲間がいるチーム戦との
  自然な差として設計した)

## 背景

- ユーザー要望:「マッチング待機は自分1人でも開始できるように」「ボスの攻撃でHP0に
  なったら一定時間操作不能、全員HP0でゲームオーバー」
- 「一定時間操作不能→復帰」という一時ダウン制は協力プレイ向けの一般的な設計
  (誰か1人が落ちてもチームが続けられる)。ソロプレイでこの仕組みをそのまま適用すると
  「全員」が自分だけなので即Defeat判定と衝突するが、これは意図的な帰結として採用した
  (ソロで落ちたら仲間がフォローできないので即座に試合終了するのは自然な挙動)

## トレードオフ

- 操作不能中でもボスの反撃対象選択(巡回)は変えていない。既に操作不能な
  プレイヤーへ追加でダメージが入ってもHPはさらに負の方向に増えるだけで、
  ゲーム進行には影響しない(復帰時は固定でReviveHpにリセットするため)
- 「誰が部屋作成者か」を追跡しない設計(D-056)のまま、`start` は接続人数が
  1人でも実行できる冪等な操作として扱う。人数チェックは撤廃していないが
  下限を1に緩めただけ
