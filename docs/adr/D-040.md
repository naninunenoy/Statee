# D-040: 時間制御を pause / resume から freeze / unfreeze に改名

## 決定

Statee の時間制御(D-026)の語彙を全面的に改名する。

- `TimeControl.Pause()` / `Resume()` / `IsPaused` → `Freeze()` / `Unfreeze()` / `IsFrozen`
- 標準コマンド `pause` / `resume` → `freeze` / `unfreeze`(`step` / `wait` は据え置き)
- SuikaGame の `game/board` State のフィールド `Paused` → `Frozen`
- 旧名との後方互換(エイリアス)は持たない(GUIDELINE の「後方互換は求められない限り維持しない」)

## 背景

- D-037 / D-039 でゲーム内ポーズ(`GamePhase.Paused`、`PauseGameCommand` /
  `ResumeGameCommand`)が入り、「Statee が時間を凍結する」と「ゲームが自身をポーズする」
  という別物が同じ pause という語で衝突した。実際に Main.cs では
  `GetTree().Paused = TimeControl の凍結 || ゲーム内ポーズ` と2要因が合流しており、
  会話・ログ・State の読解で取り違えやすい
- freeze は「外部(AI)がフレームを凍結し、step で1フレームずつ融かして進める」という
  時間制御の性質をよく表す。ゲーム側の pause 語彙(Godot の `SceneTree.Paused` 含む)と
  綴りが重ならない

## トレードオフ

- (+) `game/board` の `Frozen` と `game/scene` の `Paused`(フェーズ)が別概念だと
  名前だけで判別できる
- (−) 過去の ADR(D-003, D-018, D-023, D-026, D-027)や旧ログには pause / resume 表記が
  残る。ADR は歴史的記録として書き換えず、D-026 の状態欄に改名の注記のみ追加した
- (−) Godot 側の機構名は `SceneTree.Paused` のままなので、Godot 層のコードでは
  freeze(Statee)→ Paused(Godot)への写しが1箇所残る(Main.cs の `_Process`)
