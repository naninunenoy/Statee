# D-069 本気のゲーム開発は games/ 配下に自己完結ディレクトリで置く

## 決定

デモではない「本気で開発するゲーム」は、`samples/` とは別のトップレベル `games/` に置く。
1ゲーム = 1ディレクトリ(例: `games/FirstGame/`)で、Logic / Godot / tests / slnx を
その中に**自己完結**させる:

```
games/FirstGame/
├─ FirstGame.slnx
├─ FirstGame.Logic/
├─ FirstGame.Godot/
└─ tests/FirstGame.Logic.Tests/
```

フレームワークへは ProjectReference(`../../src/` 等)で直接依存する。
第1号として FirstGame を新設した。

## 背景

- ゲーム開発を通じて Statee に手を入れる(=フレームワークを育てる)開発スタイルは
  NuGet 参照では回らず、monorepo 側の役割(D-067 のトレードオフに明記)
- `samples/` は「動くデモ・参照実装」(docs/USING.md)であり、本気のゲームとは役割が違う
- テストを共通 `tests/` でなくゲーム内に置くのは、フレームワークが安定した後に
  ゲームを別リポジトリへ切り出して NuGet 参照(D-067)へ移行しやすくするため

## トレードオフ

- テストの置き場がフレームワーク・サンプル(共通 `tests/`)とゲーム(内包)で二流儀になる
- `tools/build-all.ps1` の slnx 探索(-Depth 2)の範囲に収まる深さを保つ必要がある
- ゲーム固有の要求から Statee を変えるときは D-013 を守り、汎用化した形で
  `src/` に入れて ADR に記録する(samples が果たしてきた役割と同じ回し方)
