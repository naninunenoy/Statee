# D-060 Declaree にフォーム系ノードを追加(CheckBox/Slider/Stack/Overlay/ReorderList/FontSize)

## 決定

宣言的 UI(Declaree、D-035)に、フォーム中心のアプリを表現するための汎用ノードを追加する。

- **CheckBox**: `CheckBox(Text, OnToggle) { Checked }`。Button と同じくイベント ID 方式。
  チェック状態は宣言(`Checked`)が正であり、押下は通知のみ → ホストが状態を変えて再構築する
- **Slider**: `Slider(Min, Max, Value, OnChange) { Step }`(Godot HSlider)。
  値はイベントに載せず、ホストが Name(D-038)で Godot コントロールを直接参照して読む
  (LineEdit と同じ方針)。~~dispatch は確定タイミングのみ(ドラッグ終了・キー操作)~~
  → D-061 で差分適用を導入し、ドラッグ中もライブ通知(ValueChanged ごと)に変更した
- **Stack**: 子を同じ領域に重ねるコンテナ(後の子が手前。素の Control + 全面アンカー)
- **Overlay**: モーダルの幕。半透明 ColorRect + `MouseFilter.Stop` で背面 UI への
  マウス入力を遮断する。子(ダイアログ本体)は幕の上に親いっぱいで載る
- **ReorderList**: ドラッグ並び替えできる縦リスト。子の上で押下 → 縦に閾値(8px)以上
  動かして離すと `"{OnReorder}:{fromIndex}:{toIndex}"` を dispatch する。
  イベント ID は文字列のままパラメータを埋め込み、IR のシリアライズ可能性(D-035)を保つ。
  閾値未満の単純クリックはドラッグにならず子(Button 等)へそのまま届く
- **UiNode.FontSize**: 全ノード共通のフォントサイズ(テーマオーバーライド)。
  スライダーで全体の文字サイズを変える、といった全ツリー再レンダーの負荷用途を想定

## 背景

- TodoApp(6つ目のサンプル。UI 集約型・非ゲーム)が要求する UI 部品
  (完了トグル・スライダー・削除確認モーダル・ドラッグ並び替え)が Declaree に無かった。
  Declaree を育てることが TodoApp の主目的なので、ノードは TodoApp 固有概念を含まない
  汎用品として declaree/ に置く(D-013 と同型の境界)
- UiSnapshot(幾何採取)は「IR の子 = Control の子(同順)」の並行走査を前提にする。
  新ノードのレンダラはドラッグハンドル等の**中間 Control を挟まない**ことを守った
  (ReorderList は VBoxContainer サブクラスの `_Input` 観測で実現)

## トレードオフ

- ReorderList のドラッグ検知は `_GuiInput` でなく `_Input`(グローバル観測)。
  子ボタンがイベントを消費しても並び替えが成立する反面、リスト外で始まったドラッグの
  誤爆は「押下位置が子の上か」で判定して防ぐ。イベントは消費しないため、
  ドラッグ後の release で子 Button の click が発火し得る(Godot の Button は
  press/release とも自分の上のときだけ発火するので実害は出にくい)
- Slider の値は再構築ごとに宣言値へ戻る。ホストは dispatch を受けたら必ず
  ロジックへ反映して再構築する(往復を怠ると見た目と状態が乖離する)
- ドラッグ中の視覚フィードバック(浮き上がり・挿入位置表示)は未実装。
  検証(Statee)観点では from/to の確定だけで十分なため見送った
