# D-071 効果音生成ツール wavee を tools/wavee に置く

## 決定

効果音をテキスト定義(`.sfx.txt`: 1 行 1 パラメタの key: value)から WAV に変換する
コンソールアプリ **wavee** を `tools/wavee/` に新設する。スコープは
「テキスト → WAV 変換器」に留める(再生・波形表示・BGM シーケンスは必要になるまで作らない)。

- パラメタは sfxr 系の最小セット: wave(square/saw/triangle/sine/noise)/ duration /
  freq / freq-slide / attack / decay / volume / duty / seed / sample-rate
- 合成は位相累積 + 線形エンベロープ。noise は位相一周ごとに引き直すサンプル&ホールド型で、
  乱数は xorshift32 自前実装(.NET のバージョンに依らず決定論)
- WAV は依存なしの自前エンコード(RIFF・16bit PCM・モノラル・無圧縮)。OS 依存なし
- 音の定義本体は各ゲーム側(例: `games/MessBreak/audio/*.sfx.txt`)に置き、
  生成した .wav も資産としてコミットする(Godot が直接読む)

## 背景

- AI エージェントは音声を直接聞けないが、レトロ系効果音は少数の数値パラメタで決まるので
  テキストで表現・調整できる(dotee = D-070 と同じ構図)
- WAV は 44 バイトヘッダ + PCM 列だけで書けるため、PNG(zlib/CRC32 が必要)より簡単。
  Godot 4 は WAV をネイティブ再生でき、効果音用途に追加要件がない
- MP3/OGG はエンコーダの外部依存が必要になるため採らない(効果音に不要)

## トレードオフ

- 無圧縮 WAV はファイルが大きいが、効果音の長さ(1 秒未満)では無視できる
- 再生確認はツール単体ではできず、Godot(AudioStreamWav)側で行う前提
- slnx が 1 本増え、`tools/build-all.ps1` の対象になる(-Depth 2 の範囲内)
