# D-066 環境依存パスを排除し Windows / macOS の両方で開発できるようにする

## 決定

OS 依存・マシン依存の記述をリポジトリの共有物から排除する:

1. **Godot のパスは環境変数 `GODOT_BIN`** にフルパスで設定する。
   CLAUDE.md・skill(verify)・エージェント(statee-checker)・tools のスクリプトは
   すべて `GODOT_BIN` を参照し、絶対パスをドキュメントにハードコードしない
   (Windows: `*_console.exe` / macOS: `Godot_mono.app/Contents/MacOS/Godot`)
2. **`.mcp.json` は `dotnet` + dll 起動**にする
   (`command: "dotnet"`, `args: [".../Statee.Mcp.dll"]`)。
   `STATEE_CLI` も `.dll` を指し、`StateeCliTool` は `.dll` 指定を検出したら
   `dotnet <dll>` で起動する(apphost の拡張子差 `.exe` / 拡張子なし を吸収)
3. **tools の PowerShell スクリプトには sh 版を併設**する
   (`build-all.sh` / `run-reversi-network.sh`)。hooks は元から sh 版
   (`format-cs.sh`)を使っており対応済み

## 背景

CLAUDE.md「環境の知識」に Godot の絶対パス(`C:\Users\...`)が、`.mcp.json` に
`.exe` がハードコードされており、macOS で開発を始めると両方が壊れる。
CLAUDE.md は「環境依存。新しい環境では書き換える」と注記していたが、
書き換え方式はマシンごとに git 差分が発生し、コミット事故のもとになる。
環境変数に逃がせば共有物は不変になり、セットアップは
「`GODOT_BIN` を設定する」の1手順に収まる。

## トレードオフ

- `GODOT_BIN` 未設定だと起動時まで気づけない。スクリプトは未設定時に
  明示的なエラーメッセージで落とし、verify skill は未設定ならユーザーに設定を依頼する
- MCP の dll 起動は `dotnet` プロセスが1枚挟まる(起動コストは無視できる範囲)
- ps1 と sh の二重管理が発生する。tools のスクリプトは小さく保ち、
  ロジックが増えるならどちらかに寄せることを検討する
