# D-042: GameOver 画面と[タイトルへ]

## 決定

1. フェーズに `GameOver` を追加する。遷移規則は GameFlow が持つ(D-031, D-037 と同型):
   `EndGame`(Playing → GameOver)/ `GoToTitle`(GameOver → Title)。
   ゲームオーバー中のポーズ・再開・やり直しは不成立(遷移を定義しない)
2. ゲームオーバーの検知は Godot 層が `SuikaLogic.IsGameOver` の true 遷移を購読して
   `EndGame()` を呼ぶ。規則(溢れ判定・猶予時間)は従来どおり SuikaGame.Logic が持ち(D-024)、
   GameFlow は「プレイ中だったら GameOver 画面へ」という画面遷移だけを持つ
3. GameOver 画面は Declaree で宣言する:「GameOver」ラベル+最終スコア+[タイトルへ]。
   [タイトルへ] → `GoToTitleCommand`(D-032 の配線と同型)
4. タイトルへ戻るときの盤面・スコアのリセットは、やり直し(D-037)と同じ
   `RestartRequests` 通知を再利用する(Godot 層の購読は1本のまま)

## 背景

- ユーザー要求: GameOver をわかりやすくする。「GameOver」と表示し、
  [タイトルへ]ボタンを表示する
- 従来はログ出力と `game/board.IsGameOver` の State 変化のみで、
  画面上はスコア表示のまま何も起きなかった

## トレードオフ

- ゲームオーバー中も物理は止めない(盤面はそのまま見え続ける)。
  凍結はロジック側の「ゲームオーバー後は入力を無視」(D-024)が担う
- `RestartRequests` の名前は「やり直し」由来だが、意味は「盤面・スコアをリセットせよ」で
  タイトル復帰にも一致するため再利用した。用途が分岐したら通知を分ける

## 状態

採用。ロジックはユニットテスト(GameFlow / Router で46件緑)、
Godot 層は headless E2E で確認済み(溢れ → Phase=GameOver、ui/tree に
GameOverLabel / FinalScoreLabel / ToTitleButton が出現、ゲームオーバー中の drop は拒否、
ToTitleButton クリックで Phase=Title・スコア0・フルーツ0、再スタート・再投下可)。
