# RaidBoss ロードマップ

第4サンプルゲーム(D-053)。2人協力ボス戦を題材に、Syncee(D-050)へ
「決定論ロックステップ」(D-054)という新しい同期パターンを実証する。

完成後はリバーシと同じ扱いとする: このファイルは削除し、要点を
docs/ARCHITECTURE.md「サンプルゲーム:RaidBoss」節に集約する。

## 進め方

### ローカル版(ネット非依存。決定論の骨格を固める)

- [x] **B-0** スケルトン(`/new-game` 雛形。GameLogic/GameState/Main のプレースホルダ、
      build/test/headless 起動確認済み)
- [x] **B-1** 失敗するテスト: プレイヤー2人・ボス1体の最小ループ
      (移動・攻撃・ボスHP減少・ボス撃破で勝利、プレイヤー全滅で敗北)
- [x] **B-2** 実装(B-1 を全緑に)
- [x] **B-3** Godot 層への接続(描画・入力・State 公開。headless で ping→step→HP確認済み)
- [x] **B-4** 決定論の検証: 同一シード+同一入力ログで2回実行し、結果(ボスHP・勝敗)が
      完全一致することをテストで担保(D-011/D-049 と同型。B-1 のテストに含めた)

### ネット版(Syncee 配線。D-054 のロックステップを実装)

- [x] **B-5** Tick 入力バンドルの確定・配布。`AuthorityLog`/`ReplicaLog`(D-050。即時確定)とは
      別に `TickBundleAuthority`/`TickReplicaLog`(D-054。バッチ確定)を Syncee コアへ追加した
      (19件全緑)
- [x] **B-6** `RaidBoss.Server`(純C#。`RaidBossAuthority`が`TickBundleAuthority`で
      両クライアントの入力を確定・配布しつつ同じ`GameLogic`をヘッドレスで並走させる。4件全緑)
- [ ] **B-7** `RaidBoss.Godot` のネットワークモード(入力遅延バッファ、確定バンドル適用)
- [ ] **B-8** マルチインスタンス E2E(D-051 のシナリオ語彙を使用。2クライアント+サーバで
      決定論が保たれること、片方切断で残り側が不戦勝相当になることを確認)

## 明示的にやらないこと

- CPU 対戦・3人以上・観戦
- 状態スナップショット配布(途中参加・再接続)
- Godot 物理エンジンを使った当たり判定(決定論を壊すリスクがあるため、
  当たり判定は `RaidBoss.Logic` 内の純粋計算で行う)
