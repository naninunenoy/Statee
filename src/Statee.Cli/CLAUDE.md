# Statee.Cli 開発指針

- **ゲーム依存のコマンドを持ち込まない**。ここは汎用 CLI で、ゲーム固有の操作は
  `send --command ...` で表現するか、ゲーム用 CLI(この CLI を包む)を別に作る(D-001)
- **出力の契約を壊さない**: payload → stdout / エラー → stderr / exit code 0・1。
  AI とスクリプトはこの契約を直接読んでいる。装飾や余計な出力を stdout に足さない
- ゲーム側の仕様変更はこの CLI の再ビルドだけで反映できる状態を保つ
  (MCP サーバーの再ビルドを要求する変更は入れない)
- トレース(STATEE_TRACE)はベストエフォート。トレースの失敗が本来の入出力に
  影響する変更を入れない(D-021)
- ワイヤプロトコルの両端は Statee.Remote とここ。片側だけ変えない(D-018)
